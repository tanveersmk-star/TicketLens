Integration patterns, agent architecture, and the technical stack behind each automation.

Agent Architecture

Each automation is a multi-step agentic pipeline: Plan → Retrieve (RAG) → Reason (LLM) → Act (Tool calls) → Verify → Store. Agents are stateful, auditable, and retry-safe. Orchestration via LangGraph or equivalent.

LangGraph / CrewAI Claude Sonnet / GPT-4o Tool calling + MCP

RAG Stack

Vector store (Chroma / Pinecone / Azure AI Search) ingests ticket exports, KB articles, SOPs, CMDB dumps, and policy documents. Hybrid retrieval (dense + sparse). Chunking strategy tuned for ITSM domain structure.

Pinecone / Chroma Hybrid retrieval ITSM-aware chunking

Integration Layer

Demo phase: file/API export ingestion. Production: webhooks or polling from ITSM APIs (ServiceNow, BMC, Jira). Agent actions write back via REST. All tool calls logged with actor, timestamp, reasoning chain for audit.

REST / Webhooks ServiceNow · BMC · Jira Full audit trail
