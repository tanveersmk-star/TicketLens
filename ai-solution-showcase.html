Security Architecture

Defence in depth.
At every layer.

Security is not an afterthought — it's woven through PDF ingestion, LLM output rendering, sentiment enforcement, and the proxy layer.

🛡️

Prompt Injection Guard

core/security.js runs secDeepScan() on the extracted PDF text before any LLM sees it. The SEC_PATTERNS[] array blocks shell execution, hidden instructions, and ClickFix-style attacks embedded in ticket content.

🔤

HTML Escaping — No Raw innerHTML

Every fragment of AI-generated content passes through esc() before DOM insertion. No AI text is ever set as raw innerHTML. XSS via LLM output is structurally impossible.

📋

Clipboard Protection

doCopy() runs the security scanner before writing any AI content to the clipboard — preventing copy-paste prompt injection chains from reaching terminal or other tools.

😤

Dual-Layer Sentiment Enforcement

Layer 1: Role-based ALLOW/DENY lists injected into every LLM system prompt. Layer 2: Client-side post-processing strips any hallucinated sentiment flags from resolver-side roles — the business rule is enforced in code, not just in the prompt.

🔌

Hardened CORS Proxy

The local proxy binds to 127.0.0.1 only, validates the Origin header, rate-limits to 30 req/min, caps request bodies at 2MB, and masks the API key in all log output.

🧱

Content Security Policy

default-src 'none' baseline. Scripts restricted to self + specific CDN origins. No inline eval. All AI API calls go through HTTPS with explicit endpoint allowlisting.
